# 美团云冰箱 Demo Script

## 目标

用 3 到 5 分钟讲清楚：

1. 家里有什么
2. 今晚吃什么
3. 还缺什么
4. 怎么补购或转外卖
5. 爸妈怎么更容易用

## 演示前准备

- 后端已启动：`http://127.0.0.1:8000`
- 前端已启动：`http://127.0.0.1:5173`
- 登录账号：`13800000000 / 12345678`

## 演示顺序

### 1. 首页开场

页面：`/`

讲法：

- 这是“美团云冰箱”，不是单纯识别工具，而是家庭库存驱动的晚饭决策中心
- 首页先给出库存总数、临期提醒、爸妈模式状态
- 用户第一眼就知道，系统不是只识别，而是能帮他把晚饭这件事做完

### 2. 拍照识别入库

页面：`/recognize`

讲法：

- 我们先拍一张冰箱里的照片
- 系统识别出番茄、鸡蛋、青椒
- 关键不是“识别出来”，而是“允许用户修正再入库”
- 这一步解决真实家庭场景里识别不准、不敢直接写库存的问题

动作：

- 选择一张图片
- 点“开始识别”
- 修改任意一个食材名称或数量
- 点“确认并入库”

### 3. 查看家庭库存

页面：`/inventory`

讲法：

- 所有确认后的食材都会进入家庭库存
- 库存页支持筛选、临期标记、手动增删改
- 这里是家庭真实库存的工作台，不是临时结果页

动作：

- 展示“临期”食材
- 搜索一个食材
- 可选演示手动新增一条库存

### 4. 今晚吃什么

页面：`/recipes`

讲法：

- 现在系统开始做真正的决策
- 推荐被清楚分成三类：
  - 不补购也能做
  - 少量补购就能做
  - 不想做饭，直接外卖替代
- 这就是从“看库存”走向“做决定”

动作：

- 先点“不补购也能做”的一个菜，比如番茄炒蛋
- 再点“少量补购就能做”的青椒牛肉
- 展示右侧详情与缺失分析

### 5. 生成补购方案

页面：`/purchase?recipeIds=rcp_beef_pepper`

讲法：

- 选中青椒牛肉后，系统知道只差牛肉
- 接下来自动生成补购方案、匹配商品，并解释为什么这样更省
- 如果今晚不想做，也能给出外卖替代

动作：

- 点“生成补购方案”
- 点“匹配商品”
- 展示商品、价格、优惠解释、外卖替代

### 6. 代付与爸妈模式

页面：`/senior`

讲法：

- 对爸妈来说，不需要复杂操作
- 爸妈模式放大字体、减少按钮、文案更直白
- 最后还能把已选好的补购方案发给孩子代付

动作：

- 开启爸妈模式
- 展示四个大按钮
- 返回补购页演示“生成下单跳转”

## 一句话总结

美团云冰箱把“家里真实库存”变成了一个可执行的生活入口：

- 先知道有什么
- 再决定吃什么
- 再判断缺什么
- 最后快速补购、转外卖，或者让家人代付
